什么是 XTFtpServer？

XTFtpServer 是一款专为 .NET 开发者设计的 FTP 服务器库，旨在简化 FTP 服务的集成流程。无需深入理解复杂的 FTP 协议细节，通过几行代码即可在您的应用中嵌入完整的 FTP 服务，支持文件上传、下载、目录管理等核心功能。

库采用异步非阻塞架构设计，确保在高并发场景下仍能保持稳定性能，同时提供灵活的扩展接口，支持自定义用户认证、权限控制和日志记录，满足企业级应用的定制需求。

适用场景

  • 桌面应用嵌入 FTP 服务，实现设备间文件共享
  • 企业内部系统的文件传输模块，替代第三方 FTP 工具
  • 测试环境快速搭建临时 FTP 服务，用于数据交换
  • 物联网设备的文件上传服务，如摄像头、传感器数据传输

安装指南

通过 NuGet 安装

使用 NuGet 包管理器或 .NET CLI 快速安装 XTFtpServer 库：

// .NET CLI 命令

dotnet add package XTFtpServer


// Package Manager 命令

Install-Package XTFtpServer

从 GitHub 克隆源码，手动编译并引用：

git clone https://github.com/prism3dcn/XTFtpServer.git


// 编译项目

cd XTFtpServer

dotnet build

快速入门示例

Program.cs (XTFtpServer 示例) 
using System;
using System.Threading.Tasks;
using XTFtpServer; // 引用 XTFtpServer 命名空间
using XTFtpServer.Authentication;

namespace XTFtpDemo
{
    class Program
    {
        static async Task Main(string[] args)
        {
            // 1. 创建用户认证服务（支持自定义扩展）
            var authService = new FtpUserAuthenticationService();
            // 添加普通用户（只读权限）
            authService.AddUser(
                username: "testuser", 
                password: "Test@123", 
                rootPath: @"D:\FTP\test", 
                permissions: FtpUserPermissions.ReadOnly
            );
            // 添加管理员用户（读写权限）
            authService.AddUser(
                username: "admin", 
                password: "Admin@123", 
                rootPath: @"D:\FTP\admin", 
                permissions: FtpUserPermissions.ReadWrite
            );

            // 2. 配置 FTP 服务器选项
            var serverOptions = new FtpServerOptions
            {
                Port = 21, // FTP 标准端口
                MaxConcurrentConnections = 50, // 最大并发连接数
                EnablePassiveMode = true, // 启用被动模式
                PassivePortRange = new Range(50000, 50100), // 被动模式端口范围
                LogLevel = FtpLogLevel.Info // 日志级别
            };

            // 3. 创建 XTFtpServer 实例
            using var ftpServer = new XTFtpServer(authService, serverOptions);

            // 4. 注册事件（日志、连接状态）
            ftpServer.LogReceived += (sender, e) => 
                Console.WriteLine($"[{e.LogLevel}] {DateTime.Now:HH:mm:ss} - {e.Message}");
            
            ftpServer.ClientConnected += (sender, e) => 
                Console.WriteLine($"客户端连接: {e.ClientIp}:{e.ClientPort}");
            
            ftpServer.ClientDisconnected += (sender, e) => 
                Console.WriteLine($"客户端断开: {e.ClientIp}:{e.ClientPort}");

            // 5. 启动服务器
            Console.WriteLine("正在启动 XTFtpServer...");
            await ftpServer.StartAsync();
            Console.WriteLine($"服务器已启动，监听端口: {serverOptions.Port}");
            Console.WriteLine("按任意键停止服务器...\n");

            // 等待用户输入
            Console.ReadKey();

            // 6. 停止服务器（释放资源）
            Console.WriteLine("\n正在停止 XTFtpServer...");
            await ftpServer.StopAsync();
            Console.WriteLine("服务器已停止");
        }
    }
}

示例说明

  • 通过 FtpUserAuthenticationService 实现用户管理，支持权限细分
  • 通过 FtpServerOptions 配置服务器参数，如端口、并发数、被动模式
  • 提供丰富的事件（日志、连接/断开），便于业务扩展和监控
  • 使用 using 语句确保服务器资源自动释放

核心特性

完整 FTP 协议支持

支持 FTP 标准命令集，包括主动模式（PORT）、被动模式（PASV），以及文件上传（STOR）、下载（RETR）、目录操作（MKD/RMD/LIST）等核心功能。

灵活的认证体系

内置用户密码认证，支持自定义认证扩展（如对接数据库、LDAP），同时提供基于用户的权限控制（只读/读写）和目录隔离。

异步高性能

基于 .NET 异步 IO 模型设计，非阻塞处理客户端连接，支持高并发访问，单服务器可稳定处理数百个并发连接。

详细日志与监控

支持多级别日志（Debug/Info/Warn/Error），可记录客户端连接、文件操作、错误信息等，同时提供事件接口便于集成监控系统。

跨 .NET 框架支持

兼容 .NET Framework 4.8+ 和 .NET 6+，可无缝集成到传统 Windows 桌面应用和现代 .NET 跨平台应用中。

高度可扩展

提供丰富的扩展接口，支持自定义文件系统（如对接云存储）、自定义命令处理、自定义日志输出，满足复杂业务场景需求。
